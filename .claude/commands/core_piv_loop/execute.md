# Execute: 按计划实现

## 要执行的计划

读取计划文件：`$ARGUMENTS`

## 执行指令

### 1. 阅读理解

- 仔细阅读整个计划
- 理解所有任务及依赖
- 记录验证命令
- 了解测试策略

### 2. 按顺序执行任务

对"逐步任务"中的每个任务：

**a. 定位任务**
- 识别文件和所需操作
- 如修改则先读取相关文件

**b. 实现任务**
- 严格按规格执行
- 保持与现有代码模式一致
- 包含类型提示和文档
- 适当添加结构化日志

**c. 边做边验证**
- 每次文件更改后检查语法
- 确保导入正确
- 验证类型定义

### 3. 实现测试策略

完成实现任务后：
- 创建计划中指定的测试文件
- 实现所有测试用例
- 确保覆盖边界情况

### 4. 运行验证命令

按顺序执行计划中的所有验证命令。

如有失败：
- 修复问题
- 重新运行
- 通过后继续

### 5. 最终验证

- ✅ 计划中所有任务完成
- ✅ 所有测试创建并通过
- ✅ 所有验证命令通过
- ✅ 代码遵循项目约定

- 测试结果
