# Prime: 加载项目上下文

## 目标

通过分析结构、文档和关键文件，全面理解代码库。

## 流程

### 1. 分析项目结构

列出所有跟踪文件：
!`git ls-files`

显示目录结构：
`tree -L 3 -I 'node_modules|__pycache__|.git|dist'`

### 2. 阅读核心文档

- CLAUDE.md 或类似规则文件
- 项目根目录和主要目录的 README
- 架构文档

### 3. 识别关键文件

根据结构，识别并阅读：
- 主入口（main.py, index.ts, app.py）
- 配置文件（pyproject.toml, package.json）
- 模型/模式定义
- 重要服务或控制器

### 4. 了解当前状态

```bash
git log -10 --oneline
git status
```

## 输出报告

### 项目概览
- 用途和类型
- 主要技术框架
- 当前版本/状态

### 架构
- 整体结构
- 关键架构模式
- 重要目录及用途

### 技术栈
- 语言和版本
- 框架和主要库
- 构建工具
- 测试框架

### 核心原则
- 代码风格约定
- 文档标准
- 测试方法

### 当前状态
- 活跃分支
- 近期变更
- 即时观察

**使用要点和清晰标题，便于快速浏览。**
