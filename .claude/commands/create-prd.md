# 创建 PRD

根据当前对话上下文生成产品需求文档。

## 输出文件

写入：`$ARGUMENTS`（默认：`PRD.md`）

## PRD 结构

### 必需章节

1. **概述** - 产品概览、核心价值、MVP 目标
2. **使命** - 使命宣言、核心原则（3-5条）
3. **目标用户** - 用户画像、技术水平、需求痛点
4. **MVP 范围** - 包含（✅）/ 不包含（❌），按类别分组
5. **用户故事** - "作为...我想...以便..."格式，含具体示例
6. **架构** - 高层架构、目录结构、设计模式
7. **功能详情** - 详细功能规格
8. **技术栈** - 后端/前端技术和版本
9. **安全配置** - 认证、配置管理、部署考虑
10. **API 规范** - 端点、请求/响应格式、示例
11. **成功标准** - MVP 定义、功能要求（✅）、质量指标
12. **实现阶段** - 分 3-4 阶段，含目标、交付物、验证标准
13. **未来考虑** - MVP 后增强、集成机会
14. **风险缓解** - 3-5 个关键风险及缓解策略

## 执行步骤

### 1. 提取需求
- 回顾对话历史
- 识别显式需求和隐式需求
- 记录技术约束和偏好

### 2. 综合信息
- 组织到适当章节
- 填补合理假设
- 确保技术可行性

### 3. 编写 PRD
- 清晰专业语言
- 具体示例
- Markdown 格式

### 4. 质量检查
- ✅ 章节完整
- ✅ 用户故事清晰
- ✅ 范围合理
- ✅ 成功标准可衡量

## 输出确认

创建后：
1. 确认文件路径
2. 简要总结内容
3. 说明假设
4. 建议下一步
